Brief description:

The log viewer displays the 50 most recent log messages generated by the rig client ordered by the most recent logs first. A color scheme is used to identify logs from a specific log level (provided the log level is contained in the log message string).

Log levels:

The rig client uses the following log levels:
  • FATAL - Messages that are generated by events that cause unexpected program termination.
  • PRIORITY - Messages that are not errors but should always be printed.
  • ERROR - Messages that are generated by events that cause unexpected results and stop program execution sequence.
  • WARN - Messages that are generated by events that cause undesired results but do not stop the expected program execution sequence.
    HINT: 'WARN' is the most appropriate level during normal operation of the rig client as it prints messages that should be attended to and is not overly verbose.
  • INFO - Messages that are generated by events that are regular in occurrence and highlight the progress of the rig client at a coarse-grained level.
  • DEBUG - Messages specifying program fine-grained execution and progression and are useful to debug rig client errors.

Log types:

The log type specifies where logs are persisted to. The rig client can be configured with the following logs:
  • RolledFile - Saves log messages to a configured text file. The text file is 'rolled' when it reaches a configured size to stop file system bloat.
    HINT: 'RolledFile' is usually the most appropriate log type during normal operation of the rig client as the log messages are easy to transport for offline anaylsis. Also, most support requests to the Sahara developers require logs as a pre-requiste for providing help.
  • File - Saves log messages to a configured text file. The log file is not 'rolled' and left unchecked is susceptible to bloating the file system.
  • Syslog - Provides the log messages to a 'syslog' server. This log type is only appropriate to UNIX operating systems.
  • WinEvents - Provides the log messages to host operating systems Windows Event Log. These may be viewed by opening 'Control Panel' -> ''Administrative Tools' -> 'Event Viewer'. This log type is only appropriate to Windows operating systems.
    NOTE: Before the WinEvents log type can be used the file NTEventLogAppender.dll file must be placed in the rig client's library path. Otherwise this log type will fail with a UnsatisfiedLinkError.
  • SystemErr - Prints the log messagse to the standard error stream of the rig client process. With this log type log messages are not persisted but are lost after they are printed.

Log format:

The log format dictates what is displayed as a log message and can be configured system wide or on a per log level basis. The configuration involves using 'macro' strings which are place holders to be replaced by the run time value.
  • __MESSAGE__ - Actual log message.
  • __LEVEL__ - Level the message was logged at (i.e. ERROR, WARN, ...).
  • __TIME__ - Message generation time with hour of day:minute:second.
  • __DATE__ - Message generation date with day:minute:year.
  • __ISO8601__ - Message generation timestamp in the ISO8601 format.
  • __CLASS__ - Class where the message orginated.
  • __METHOD__ - Method where the message orginated.
  • __SOURCE__ - Source file where the message orginated.
  • __LINE_NUM__ - Line number where the message orginated.
  • __TID__ - Thread id which executed the instruction to log the message.
